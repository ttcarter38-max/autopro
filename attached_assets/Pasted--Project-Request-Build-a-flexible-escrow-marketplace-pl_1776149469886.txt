 Project Request
Build a flexible escrow marketplace platform called SuperCarTrust with the following specifications:

Core Concept
A secure bank transfer-based escrow platform for high-value items (vehicles, real estate, art, collectibles). Features a professional AutoPro-inspired automotive dealership design with a clean white/light background, green/white/blue color scheme similar to Escrow.com. Includes a full-width hero image slider, premium car listings, a dedicated "How Escrow Works" explanation page, and a complete 5-step escrow transaction workflow.

Revolutionary Features
No Seller Registration Required - Sellers receive email notifications with secure token links to accept/reject transactions without creating accounts
Manual Product Entry - Buyers manually enter any product details (not limited to predefined items)
Password-Protected Seller Actions - Seller acceptance form requires password: crocodilu123
Unique Bank Accounts per Transaction - Each seller provides their own bank details when accepting
Payment Proof Upload - Buyers can upload documents (receipts, screenshots) when confirming payment
Complete Email Notification Chain
1. Buyer Initiates Transaction:

Seller receives "New Escrow Transaction" email
Includes: product details, buyer info (name, email, address), amount
Two buttons: "Accept & Enter Bank Details" and "Reject Transaction"
Uses secure token links (no login required)
2. Seller Accepts:

Buyer receives "Payment Instructions" email
Includes: seller's unique bank account details (bank name, account name, account number, routing number, SWIFT code)
Reference number and exact amount to transfer
Instructions to upload payment proof
3. Buyer Confirms Payment:

Admin receives "Payment Confirmation Awaiting Verification" email with transaction details and bank reference
Seller receives "Payment Received" email notifying them buyer has paid, admin verification in progress
4. Admin Releases Funds:

Seller receives "Funds Released - Transaction Completed" email
Includes: final summary, bank account confirmation, delivery coordination details with buyer contact info
Transaction Flow
Transaction Statuses:

Pending Seller Acceptance - Initial state
Awaiting Payment - Seller accepted with bank details
Payment Submitted - Buyer confirmed with optional payment proof
Completed - Admin verified and released funds
Rejected - Seller rejected

Important Implementation Details
Seller Acceptance Flow:

GET /escrow/accept/<id>?token=<token> verifies token, shows password form
POST password → verify crocodilu123, store session['seller_verified'] = transaction.id
Redirect back to GET → now show bank details form
POST bank details → update transaction, status='Awaiting Payment', send buyer email
Payment Confirmation:

Buyer uploads optional payment proof file
Save with secure filename in uploads/ directory
Update transaction: bank_ref, payment_proof_file (if uploaded), status='Payment Submitted'
Send notifications to both admin and seller
Admin Release:

Verify status is 'Payment Submitted'
Update status to 'Completed'
Send completion email to seller with delivery coordination info
Token Security:

All seller action links include ?token=<secure_token>
Always verify token matches transaction.secure_token before proceeding
Return 403 if token mismatch

Include action buttons with full URLs
